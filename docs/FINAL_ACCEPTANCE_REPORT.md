## 2. 本轮修复

### 安全与权限

- 新增医生必须显式填写 6–72 位初始密码，取消固定默认密码；用户名限制为 3–32 位字母、数字、下划线或短横线。
- 按预约查询病历时先校验预约归属；即使病历尚未创建，也不会向其他患者泄露状态。
- 开发环境未设置 JWT 密钥时使用每次启动自动生成的临时密钥；生产 profile 继续强制 `APP_JWT_SECRET`。
- 浏览器 Cookie 写操作启用双提交 CSRF Token；缺少令牌返回中文 403，Bearer API 客户端不受影响。
- 生产 profile 强制禁用演示数据初始化，空生产数据库不会创建固定演示账号和演示排班。

### 业务一致性

- 已有预约的排班不允许更换医生、日期或时段，避免历史预约被后台误改。
- 排班修改到已存在时段时在业务层返回中文 400，不再由数据库抛出 500。

### UI 与可用性

- 移除没有实际行为的顶部通知按钮，避免误导用户。
- 新增医生表单补齐密码类型、必填、长度、用户名格式和中文提示。
- Windows 前端启动脚本支持 `-BindAddress` 和 `-Port`，端口冲突时可稳定启动到 5188。
- 运行时界面和源码用户文案均为简体中文；状态码等内部枚举通过中文标签展示。
