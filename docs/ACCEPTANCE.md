# 成品验收清单

更新时间：2026-07-12

## 结论口径

本项目已达到**本科毕业设计可演示成品**标准，并补齐了三项增强：

- MySQL + Docker 一键启动（可选）
- 核心接口自动化测试
- 答辩演示脚本与启动脚本

- 前后端分离可本地运行
- 三角色主流程可闭环
- 有真实后端鉴权与持久化
- 有 README / 启动方式 / 演示账号

尚未达到**生产上线级成品**标准（无完整 CI/CD、无公有云部署、无支付短信等业务扩展）。

## 已知限制

1. 默认数据库为 H2 文件库，适合演示；生产建议切 MySQL  
2. 无短信/支付/医保/住院等扩展业务  
3. 自动化测试覆盖核心路径，非全量回归  
4. Docker Compose 当前只包含 MySQL，不含前后端容器  

## 演示脚本

见 `docs/DEFENSE_DEMO.md`，或：

1. `powershell -File scripts/start-backend.ps1`  
2. `powershell -File scripts/start-frontend.ps1`  
3. patient 预约 → doctor 写病历 → patient 看病历 → admin 看板
